Flask Web-抄书(1)

背景环境：32位win10系统

第一章 安装

1.安装虚拟环境

1.1 前提：虚拟环境由第三方工具virtualenv创建

1.2 步骤：

  • python中安装virtualenv工具，在cmd环境中执行pip install virtualenv命令
  • 在新文件夹中创建虚拟环境：virtualenv venv ,venv是安装了全新的虚拟环境文件夹。
  • 使用前激活虚拟环境：venv\Scripts\activata,如果取消则相应的执行deacitvate.
  • 激活后显示：(venv)$

2.安装flask扩展包

2.1 安装flask：

(venv)$ pip install flask

验证flask是否安装正确，在python解释器中导入flask：

(venv)$ python
>>> improt flask
>>>

更新flask：

(venv)d:\learnfla> easy_install -U Flask

第二章 程序的基本结构

2.1 初始化

创建程序实例(Flask程序必须创建一个程序实例，程序实例就是Flask类的对象)

from flask import Flask
app=Flask(__name__)

Flask类的构造函数(init())只有一个必须指定的参数，即程序主模块或包的名字。在大多数程序中，Python的____ name __ __变量就是所需的值。

2.2 路由和视图函数

2.2.1 路由-含义：处理URL和函数之间关系的程序被称为路由。（客户端即浏览器发送请求到web服务器，web服务器再把请求发送给Flask程序实例。程序实例需要知道对每个URL请求运行哪些代码，所以保存了一个URL到Python函数的映射关系。）

2.2.2 路由-定义：使用程序实例提供的’app.route’装饰器，把修饰函数注册为路由。

使用：

@app.route('/')
def index(x):  # 此函数注册为程序根地址的处理程序
    return '<h1>hellow world</h1>'  # 返回值称为‘响应’，是客户端接受的内容。

2.2.3 视图函数(view function):像index()这样的函数成为视图函数。视图函数返回的快车是HTML字符串，也可以是复杂的表单。

注：在python代码中嵌入响应字符串会使代码难以维护。

Flask支持URL格式：很多地址中包含的可变部分,只需在route修饰器中使用特殊的句法即可。

@app.route('/usr/<name>')   # <name>就是动态部分
def user(name):
    return '<h1>hello,%s!</h1>' %name

注：路由中的动态部分默认使用字符串，但也可使用类型定义。支持int、float、path等类型。path类型也是字符串，但不把斜线视为分隔符，而将其当做动态片段的一部分。

2.3 启动服务器

程序实例使用run方法启动Flask集成的开发web服务器：

if __name__=='__main__':
    app.run(debug=True)

app.run()函数接受用于设置web服务器的操作模式。启用调试模式可以为工作带来便利，比如激活调试器和重载程序。把debug参数设为True，启用调试模式。

2.4 一个完整的程序

2.4.1 示例，hello.py，一个完整的Flask程序

from flask import Flask
app=Flask(__name__)

@app.route('/')
def index(x):
    return '<h1> hello,world</h1>'

if __name__==__main__:
    app.run(debug=True)

2.5 请求-响应循环

2.5.1 程序/请求上下文

Flask从客户端收到请求时，要让视图函数能访问一些对象，这样才能处理请求。 为了避免可有可无的参数把视图函数弄得一团糟，Flask使用上下文临时把某些对象变为全局可访问。

有了上下文的视图函数：

from flask import Flask

@app.route('/')
def index():
    user_agent=request.headers.get('User-agent')
    return '<p> Your brower is %s<p> %user_agent

在Flask中，有两种上下文：程序上下文和请求上下文。下表中列出了这两种上下文提供的变量。

变量名上下文说明
current_app程序上下文当前激活程序的程序实例
g程序上下文处理请求时用作临时存储的对象。每次请求都会重设这个变量
request请求上下文请求对象，封装了客户端发出的HTTP请求中的内容
session请求上下文用户会话，用于存储请求之间需要‘记住’的值的词典

Flask在分发请求前激活/推送程序/请求上下文，请求处理完成后再将其删除。如果使用变量时没有激活程序/请求上下文，就会导致错误。

程序上下文的使用方法：

>>> from hello improt app
>>> from flask import current_app
>>> current_app.name
Traceback (most recent call last):
···
RuntimeError:working outside of application context
>>> app_ctx=app.app_context() # 在程序实例上调用app.app_context()可获得一个程序上下文
>>> app_ctx.push()
>>> current_app.name
'hello'
>>> app_ctx.pop()

在这个例子中，没激活程序上下文就调用current_app.name会导致错误，但推送完上下文后就可以调用了。

2.5.2 请求调度

Flask程序在收到客户端发来的请求时，要找到处理该请求的视图函数。为了完成这个任务，Flask会在程序的URL映射中查找请求的URL。URL映射是URL和视图函数之间的对应关系。Flask使用app.route修饰器或非修饰器形式的app.add_url_rule()生成映射。

查看Flask程序中URL映射，可检查hello.py生成的映射。

(venv) D:\learnfla>python
>>> from hello import app
>>> app.url_map
Map([Rule '/' (HEAD,OPTIONS,GET)    -> index>,
 <Rule '/static/<filename>' (HEAD,OPTIONS,GET) ->static>,
 <Rule '/user/<name>' (HEAD,OPTIONS,GET) ->user>])

‘/’和‘user/’路由是在程序中使用app.route修饰器定义。/static/路由是Flask添加的特殊路由，用于访问静态文件。

URL映射中的HEAD,OPTIONS,GET是请求方法，由路由进行处理。Flask为每个路由都指定了请求方法，这样不同的请求方法发送到相同的URL上时，会使用不同的视图函数进行处理。HEAD、OPTIONS方法由Flask自动处理，因此可以这么说，URL映射中的3个路由都是用GET方法。

2.5.3 请求钩子

为了避免在每个视图函数中都是用重复的代码，Flask提供了注册通用函数功能，注册的函数可在请求被分发到视图函数之前或之后调用。

请求钩子是用修饰器实现，Flask支持4种钩子：

  • before_first_request: 注册一个函数，在处理第一个请求前运行
  • before_request: 注册一个函数，在每次请求前运行
  • after_request: 注册一个函数，如果没有未处理的异常抛出，在每次请求之后运行。
  • teardown_request: 注册一个函数，即使有未处理的异常抛出，也在每次请求之后运行。

2.5.4 响应

Flask调用视图函数后，会将其返回值作为响应的内容。大多数情况下，响应就是一个简单的字符串，作为HTML页面回送客户端。

HTTP协议需要作为请求响应的字符串+状态码，Flask默认设为200，表明请求已被成功处理。

如果视图函数返回的响应需要使用不同的状态码，可以把数字代码作为第二个返回值添加在响应文本之后。 例如：

@app.route('/')
def index():
    return '<h1>Bad Request</h1>'，400

视图函数返回的响应还可以接受第三个参数，这是一个由首部(header)组成的字典，可以添加到HTTP响应中。

如果不想返回由1个、2个或3个值组成的元组，Flask视图函数还可以返回Response对象。make_response()函数可接受1个、2个、3个参数（和视图函数的返回值一样），并返回一个Response对象。 有时我们需要在视图函数中进行这种转换，然后在响应对象上调用各种方法。

例子：创建响应对象，添加cookie：

from flask import Flask

@app.route('/')
def index():
    response=makr_response('<h1>This document carries a cookie!</h1>')
    reponse.set_cookie('answer','42')
    return response

有一种名为重定向的特殊响应类型，这种类型没有文档，知识告诉浏览器一个新的地址用以加载页面。重定向经常在web表单中使用。 重定向经常使用302状态码表示，指向的地址由Location首部提供。重定向响应可以使用3个值形式的返回值生成，也可在Response对象中设定。由于使用频繁，Flask提供了redirect()辅助函数，用于生成这种响应：

from flask import redirect

@app.route('/')
def index():
    return redirect('http://www.example.com')

处理错误的响应：由abort函数生成，返回状态码404： 例子：URL中的动态参数id对应的用户不存在，返回404状态码。

from flask improt abort

@app.route('/user/<id>')
def get_user(id):
    user=load_user(id)
    if not user:
        abort(404)
    return '<h1> hello,%s</h1>' %user.name

2.6 Flask扩展

Flask被设计为可扩展形式，开发者可以自由选择最合适程序的包，或者按需求自行开发。

把扩展整合到程序中,例子使用Flask-Script支持命令行选项： Flask-Script为Flask程序添加了一个命令行解析器。

安装：(venv) D:\learnfla>pip isntall flask-script

示例，使用Flask-Script

from flask_script import Manager
from flask import Flask

app=Flask(__name__)
manager=Manager(app)

@app.route('/')
def index():
    return '<h1>hello,world</h1>'


if __name__=='__main__':
    manager.run()

这样修改之后，程序可以使用一组基本的命令行选项。现在运行’python hello.py’，会显示一个用法消息：

(venv) d:learnfla>python hello.py
usage: hello.py [-h] {shell,runserver}...

positional arguments:
  {shell,runserver}
    sehell     # 在flask应用上下文中运行python shell
    runserver  # 运行flask开发服务器：app.run()

optional arguments:
  -h, --help  # 显示帮助信息并退出

–host是一个很有用的选项，它告诉web服务器在哪个网络接口上坚挺来自客户端的连接。默认情况下，Flask开发web服务器坚挺localhost上的连接，所以只接受来自服务器所在计算机发起的连接。下述命令让web服务器坚挺公共网络接口上的连接，允许同一网络中的其他计算机连接服务器：

(venv) d:learnfla>python hello.py runserver --host:0.0.0.0
* Running on http://0.0.0.0:5000/
* Restarting with reloader

现在，web服务器可使用http://a.b.c.d:5000/网络中的任一台电脑进行访问，其中a.b.c.d是服务器所在计算机的外网ip地址。
