hugo搭建博客

December 02, 2017 hugo

这事一篇关于hugo生成博客及安装在GitHub上实现GitHub pages效果博客的文章，流程及步骤简单极易上手。现在的问题是主题下的说明文档内容较少，需要到gohugo.io查找相关的标准。

一.基础安装

1.Git及本地链接GitHub秘钥操作

2.hugo 客户端下载地址

  • 下载Windows版本(以自己电脑为主)

  • 在Windows环境变量path中加入hugo.exe的路径

  • 安装配置后，在命令行输入hugo –help 会有输出

3.GitHub账户及了解仓库(respository)的建立

thumbnail:“/img/logo/innovation.png”

二.生成博客

1.生产站点命令:hugo new site mysite

2.生产文章命令:hugo new name.md name.md文件将在文件夹content中生成

3.在文件夹内生成.md格式文章: hugo new post/name.md,文件位置将会在content/post/name.md

4.皮肤选择

  • 进入hugo themes主题网站,选择合适的主题

  • 在mysite文件夹下建立themes文件夹

  • 使用git

    • $ cd themes

    • git clone 主题地址

    • 编辑根目录下的config.toml 添加theme="主题名"

5.运行hugo（cmd中）

  • 在站点根目录下执行hugo命令: hugo server --theme=主题名
    • `--theme=主题名 可以省略
    • 还可以写作 hugo server --theme=主题名 --buildDrafts
    • –buildDrafts参数的意思是渲染所有的post，包括draft=true(默认不显示)状态的
    • 浏览器访问提示地址localhost:1313,显示站点

三.部署博客

1.站点根目录下生成最终页面

hugo - -theme=主题名 - -baseUrl=“https://name.github.io" 此时已在根目录下生成了public文件夹，我们将会提交此文件夹到GitHub上

2.git

  • cd public

  • git init

  • git remote add origin https://github.com/name/name.github.io

  • git add -A

  • git commit -m “first”

  • git push -u origin master

没什么问题就可以在浏览器访问你的GitHub page页面进行访问了
